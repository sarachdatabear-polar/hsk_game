# i18n Pass 2 — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Localize the strings pass 1 (2026-07-06) deferred — howto body, streak pill, flashcard hints, street caption, HUD control titles, shop item + building names — and land the deferred hygiene items (static-key usage test, `{min}` param, tagline key rename, `t()`-shadow cleanup).

**Architecture:** Same engine as pass 1: `src/i18n.js` STRINGS tables (EN+TH) + `t(key, params)` + `data-i18n`/`data-i18n-title` static annotation. Item/building names get per-id keys (`item.<catalog-id>`, `building.<building-id>`) resolved at render time with the English `name` field as fallback (so a missing key can never blank a label). Placeholder-parity test auto-guards new keys; a new static-usage test closes the "key referenced but missing" gap in both directions.

## Global Constraints (inherited, binding)

- Pure string tables in `src/i18n.js`; `main.js` wiring only; no new deps; file:// safe.
- EN values = the current hard-coded literals moved verbatim (no copy rewrites). TH values: natural Thai consistent with pass-1 tone; add every new key to `docs/i18n/i18n-translation-review.md` for native review.
- Placeholder params must match across locales (existing test enforces).
- 595 tests green throughout; `node scripts/responsive-sweep.mjs` 10/10 before ship (long Thai strings must not break the fold rules); SHELL bump + dist in ship commit; Co-Authored-By trailer; never commit `.superpowers/`.
