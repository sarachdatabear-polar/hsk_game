# UX/UI Audit — Lucky Cat HSK

**Date:** 2026-07-09
**Method:** Chromium (Playwright) screenshot sweep — 12 screens × 3 breakpoints (390×844 mobile, 768×1024 tablet, 1440×900 desktop) + `scores`, light theme.
**Baseline:** `.quest-row` frame + centering rules verified identical on `origin/development` **and** `origin/main` — findings below are **live**, not stale. (Working tree was on `feat/deco-png-legacy`.)
**Checklist reference:** ui-ux-pro-max pre-delivery list (contrast / focus / reduced-motion / cursor / hover / responsive breakpoints).

## Verdict

The art direction and visual identity are genuinely strong and cohesive — painted CN-village scenes, the reading-cat mascot, consistent gold/green/cream palette, a distinctive hand-drawn nine-slice frame system. The **accessibility floor is solid** (see checklist). The problems are **layout polish and asset-completeness**, plus one real rendering bug on the Quests screen.

---

## Findings (severity-ranked)

### 🔴 HIGH — Quests rows: frame ornament paints over the text — ✅ FIXED (branch `feat/ux-polish-quests-screens`)
**Fix applied:** `.quest-row` padding `6px 10px` → `8px 22px` (clears the 18px frame band: `ui-card-paper` slice 36 @ scale 2), and `.quest-row .qd` dropped `overflow/ellipsis/nowrap` so descriptions wrap instead of truncate. Verified at mobile + desktop: coins render in full, "Open" clears the bracket, descriptions wrap to 2 lines. `sw.js` SHELL bumped v36 → v37.

On `#s-quests`, the `ui-card-paper.svg` nine-slice frame overlaps the row content: the left bracket collides with the "Open" status chip, and the right bracket clips the reward so `+150 coins` renders as `+150 coin§`. Reproduces at every breakpoint (mobile + desktop confirmed).

- **Root cause (verified):** `.quest-row` (`index.html:467`) has only `padding:6px 10px` and `border:1px`, but the frame is applied at runtime via `border-image:var(--f-ui-card-paper)` (set by `src/assets.js:54` from `assets/ui-card-paper.svg`). The frame's bracket corners paint into a border box far thicker than 1px, and the tight padding reserves no room for that inset, so the ornament overlaps the flex content.
- **Proof it's the padding, not the frame:** the `scores`/"Best Sessions" card uses the *same* frame (`.scorelist`, same border-image rule) and renders its brackets cleanly — because it has generous padding and centered content.
- **Fix:** give `.quest-row` frame-aware padding (e.g. `padding:10px 18px`, matching the `.scorelist`/`.screen-card` inset) so the `.qd`/`.qp`/`.qr` content clears the bracket slice. Optionally reduce the frame `/ widths` for this smaller element.

### 🟡 MEDIUM — Shop tiles mix finished art with placeholder glyphs
In `#s-shop`, some catalog items show painted thumbnails (Beach Cat, Island Sunset) while others fall back to flat emoji/glyph icons on empty gradient tiles (Neon Cat Sign, Star Shower, Lion Dance Drum, Shaved-Ice Cart). The inconsistency reads as unfinished and breaks the otherwise-premium look; it also trips the checklist's "no emoji icons" rule.

- **Fix:** either paint the missing thumbnails or give un-arted items a consistent styled placeholder (framed silhouette in the same illustration language), so the grid reads as one system.

### 🟡 MEDIUM — Sparse secondary screens leave a large empty void — ✅ FIXED (branch `feat/ux-polish-quests-screens`)
**Fix applied:** `#s-street/#s-quests/#s-more` changed from `justify-content:center` to `flex-start`, so the header pins to the top like every other screen instead of floating mid-screen on tall phones/desktop. Verified at 390×844 and 360×640.

_Original finding:_
`Quests`, `Street`, `More` set `justify-content:center` (`index.html:57`); `scores`, `tones`, and empty-state cards are top-aligned but short. On tall/desktop viewports the header + content float in the middle (or top) of a big cream field — especially stark on **Quests** and **More**, which read as half-loaded.

- **Note:** the centering is *deliberate*, not a bug — it just backfires when content is sparse.
- **Fix options:** cap content to a top-aligned column with comfortable max-width; or fill the void intentionally (ambient scene art / the cloud motif already used in headers) so "sparse" looks composed rather than empty.

### 🟢 LOW / POLISH
- **Desktop = a phone column in a void.** At ≥1024px the whole game is a ~640px centered column on bare cream. Acceptable for a mobile-first PWA/Android title, but if web presentation matters, consider a subtle framed "device" treatment or a background scene behind the column.
- **Tone Trainer glyphs.** The tone marks (`ˉ ´ ˇ`` `) float small to the upper-right of the digits and are hard to parse as tone contours; a larger diacritic or a small tone-curve icon would read better.
- **"Open" quest chip wording.** Reads like a verb/button. `New`, a status dot, or dropping it (once the frame overlap is fixed) is clearer.
- **No dark theme.** The app is light-only (0 `prefers-color-scheme` for the shell; the top-right "☀️" is the Settings gear → More, not a theme toggle). Not a defect — a possible enhancement.
