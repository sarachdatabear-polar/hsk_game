# Lucky Cat HSK Art Assets

This folder contains the bitmap assets consumed by `index.html` and the canvas
renderer. Base assets are generated by `../make_assets.py`; premium shop assets
are generated by `../scripts/generate-shop-assets.mjs`.

## Regeneration

From `game/`:

```sh
npm run assets:shop
```

This writes the purchasable skin and backdrop PNGs:

- `cat-midnight-walk.png`
- `cat-midnight-happy.png`
- `cat-sakura-walk.png`
- `cat-sakura-happy.png`
- `cat-jade-walk.png`
- `cat-jade-happy.png`
- `cat-gold-walk.png`
- `cat-gold-happy.png`
- `cat-boss-walk.png`
- `cat-boss-happy.png`
- `bg-market.png`
- `bg-temple.png`
- `bg-bamboo.png`

The generator uses only Node built-ins, so it does not require Python or npm
packages beyond the project itself.

## Runtime Wiring

`src/sprites.js` preloads:

- Base cat sheets: `cat-walk.png`, `cat-happy.png`
- Premium skin sheets: `cat-<skin>-walk.png`, `cat-<skin>-happy.png`
- Boss sheets: `cat-boss-walk.png`, `cat-boss-happy.png`
- Premium backdrops: `bg-market.png`, `bg-temple.png`, `bg-bamboo.png`
- Shared sprites: `maneki.png`, `coin.png`

`src/cat.js` uses premium skin sheets first. If a sheet is missing or still
loading, it falls back to the base cat sheet with the old color filter, then to
the vector cat fallback.

`src/main.js` uses premium backdrop PNGs first. If a backdrop image is missing
or still loading, it falls back to the canvas-drawn backdrop.

## Replacement Rules

Higher-fidelity painted or AI-assisted assets can replace these files directly
as long as the filenames and rough dimensions stay compatible:

- Cat walk sheets: 6 horizontal frames, 256x256 each
- Cat happy sheets: 4 horizontal frames, 256x256 each
- Battle backdrops: landscape PNG, currently 1024x512

After replacement, run:

```sh
npm run build
```
